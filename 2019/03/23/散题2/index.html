Posted on 2019-03-23

[Noip2018 day1 t3] 赛道修建]

题目描述

C 城将要举办一系列的赛车比赛。在比赛前，需要在城内修建 mm 条赛道。

C 城一共有 n 个路口，这些路口编号为 1,2,…,n，有n−1 条适合于修建赛道的双向通行的道路，每条道路连接着两个路口。其中，第 i 条道路连接的两个路口编号为 ai 和 bi，该道路的长度为 li。借助这 n-1条道路，从任何一个路口出发都能到达其他所有的路口。

一条赛道是一组互不相同的道路 e1,e2,…,ek，满足可以从某个路口出发，依次经过 道路 e1,e2,…,ek（每条道路经过一次，不允许调头）到达另一个路口。一条赛道的长度等于经过的各道路的长度之和。为保证安全，要求每条道路至多被一条赛道经过。

目前赛道修建的方案尚未确定。你的任务是设计一种赛道修建的方案，使得修建的 m 条赛道中长度最小的赛道长度最大（即 m 条赛道中最短赛道的长度尽可能大）

输入输出格式

输入格式

输入文件第一行包含两个由空格分隔的正整数 n,m，分别表示路口数及需要修建的 赛道数。

接下来 n-1行，第 i 行包含三个正整数 ai, bi ,li, 表示第 i 条适合于修建赛道的道 路连接的两个路口编号及道路长度。保证任意两个路口均可通过这 n-1 条道路相互到达。每行中相邻两数之间均由一个空格分隔。

输出格式

输出共一行，包含一个整数，表示长度最小的赛道长度的最大值。

解题思路

距离noip2018过了有一段时间呢，突然回想过去的noip，真的是一段惨痛的经历，感觉自己并没有达到自己对自己的期望呢，还是得加油啊！

那么来看看这道题，出题人还是很心善的，部分分给的很全面，但是我在考场上只拿了30分 真的是菜啊。

先从题意入手，题目中要我们在整个图中找到m条路，并且使其中长度最小的最大。最小的最大，那么又是一道用二分求解的题？

开始想了这么多了，就开始讲讲思路吧，这题是一道链上二分的题，我们二分一下最小的赛道长度，每次进行一次判断，每次大于了mid就ans++，最后看看ans与m的大小比较，如果ans较大，则此时的mid并不为长度最小的(也同样可以理解为这几段中多出来的可以加在较短的段上，使mid并不为长度最小的线段)，那么答案可以继续增大，如果m较大，则取到mid的线段不足m个，则答案要减小。

下面借助几部份代码理解一下：

这一部分就是把自己子节点中的最大值更新过来，如果此时的长度大于了mid，那么现在这几段就可以拼在一起，直接ans++即可。

这一段代码就是把当前的子树之间产生的线段拼凑在一起，那么如果两个段拼在一起如果大于了mid，即可以把它合成一个段。

结合上面两部分我们可以看出这就是一个把线段拼在一起的操作，这种操作一共有两种：

1.在自己所有的子树里面找，如果有两个段可以拼在一起大于mid的，那么就ans++。

2.将每次在当前节点中把能找到的最长线段传到上一层（因为一个点只能用一次，所以只能传最长的线段），如果在上一层找到了能与此线段拼在一起的，同样ans++。

那么链上二分的操作就这么多了，而二分的上界怎么找呢，直接找直径不就好了吗？
